Etat : Le programme répond à toutes les demandes de fonctionnalités demandées avec une interface graphique de type console.

Fonctionnement : Le fichier main est : resto.cpp . Il est constitué d'une classe vide (nous ne savions pas quoi mettre dedans) et d'une fonction main
		 gérant les appels de chaque fonction, construction de classe afin de permettre à l'utilisateur d'enregistrer dans un premier temps,
		 un choix de plats grâce à l'interface de gestion de menu (possédant les outils d'ajout, de listage, de suppresion, de modification de plats) 
		 pour ensuite continuer sur un réglage de la date (utile pour l'historique). Une fois l'initialisation terminée, l'utilisateur peut ensuite
		 enregistrer des commandes, retourner sur le menu de gestion des produits, changer la date et accéder à un historique intuitif comportant
		 des statistiques de vente, le détail de chaque commande et le bénéfice (or coût de production).

Améliorations possibles : Nous avions quelques idées simples afin d'améliorer notre programme mais elles semblaient peu utiles à rajouter comme par exemple :
			 - Modifier la classe resto afin de pouvoir gérer plusieurs restaurants.
			 - Intégrer plusieurs produits dans une commande.
			 - Une interface graphique plus développée.
