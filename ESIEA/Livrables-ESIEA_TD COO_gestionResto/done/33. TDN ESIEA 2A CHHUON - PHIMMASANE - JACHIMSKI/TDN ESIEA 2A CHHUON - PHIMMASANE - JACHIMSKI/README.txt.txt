Fonctionnement : 

	Le programme est décomposé en classes et toutes les fonctionnalités sont rassemblées dans le main comme la récupération de données. 
L’utilisateur pourra s’y retrouver via la petite interface qui a été créé dans le main. 
L’utilisateur pourra stocker des commandes où les coordonnées du client,  les plats du client et le prix de la commande seront répertoriés. 
Pour ensuite, plus tard, rechercher les informations sur ses clients fidèles ou les produits phares de son restaurant. 
L’historique est la classe où toutes les commandes sont stockées dans des listes chainées.

	- Ensuite ajouter d’autres fonctionnalités pour avoir des «  grosses commandes » c’est-à-dire pouvoir entrer plus d’un plat dans une commande. 
Pour le moment, notre classe commande ne peut stocker qu’un produit à la fois. De ce fait on pourrait ajouter des fonctions telles que le prix total d’une commande.

	- La dernière amélioration possible serait d’ajouter une interface graphique de qualité pour être plus professionnel vis-à-vis de l’utilisateur.
